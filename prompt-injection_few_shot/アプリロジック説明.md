# アプリロジック説明 (Few-Shot Security & Triple Layer Defense)

このアプリケーション（`prompt-injection_few_shot`）は、LLM（Gemini）に対する **プロンプトインジェクション攻撃** を防ぎ、且つ **機密情報の漏洩** を検知・遮断するために、**3つの防御層（Triple Layer Defense）** と **カナリートークン** を組み合わせたアーキテクチャを採用しています。

## 1. 3つの防御レイヤーと技術的設計

本システムでは、役割の異なる3つの処理フェーズが、リクエストの「入口」から「出口」までを多層的に保護しています。

### 1. 入力チェック (Input Guard / インテント検知)
- **使用ロジック**: `classify_sensitive_content_cached`
- **機能**: ユーザーの入力内容を分析し、プロンプトインジェクションの「攻撃意図」を検知します。
- **セキュリティ設計**: 
    - **ハイブリッド判定**: 特定キーワード（「system prompt」「magic word」等）のパターンマッチングと、LLMによる意図解析を併用しています。
    - **役割**: あくまで「攻撃者の意図」を遮断する層であり、この段階では具体的な機密情報（カナリートークン）の有無は判定しません。
- **効果**: 悪意ある入力を入り口で弾くことで、メインAIへの攻撃を未然に防ぎ、リソース消費や履歴汚染を回避します。

### 2. メインAI (Few-Shot Security / 自律防御)
- **使用プロンプト**: `SYSTEM_PROMPT_TEMPLATE`
- **機能**: ユーザーの本来の要求に応えるメイン人格「ワナビー」を提供します。
- **セキュリティ設計**: 
    - **Few-Shotプロンプティング**: プロンプト内に「過去の攻撃パターンとそれに対する正しい拒絶回答」の例を記述しています。
    - これにより、入力ガードをすり抜けた未知の攻撃に対しても、AI自身が自律的に「ワナビー」らしく拒絶するように訓練（インコンテキスト学習）されています。

### 3. 出力チェック & 回避用AI (Output Guard / 漏洩遮断)
- **使用ロジック**: `likely_leak` / `generate_refusal_response`
- **機能**: メインAIの回答の中に、**動的カナリートークン**が含まれていないかを最終チェックします。
- **セキュリティ設計**: 
    - **文字列完全一致**: `likely_leak` は、回答テキスト内にセッション固有のカナリートークン（文字列）が存在するかのみを厳密にチェックします。
    - **回答の破棄**: トークンの混入を検知した場合、メインAIの回答はシステム内部で完全に破棄されます。
    - **安全な差し替え**: 回避用AI（Refusal Agent）が、機密を知らない安全な設定（`REFUSAL_SYSTEM_PROMPT`）に基づいてお断りメッセージを新しく生成し、それをユーザーに表示します。

---

## 2. カナリートークン (Canary Token)

- **役割**: 漏洩を確実に検知するための「動的な目印」です。
- **生成**: セッションごとに `generate_canary_token()` によって固有のランダムな文字列が発行されます。
- **検知**: `likely_leak` 関数が、AIの回答内にこの文字列がそのまま出現していないかを監視します。このシンプルな一致確認が、最も確実な漏洩検知の手段となります。

---

## 3. 処理フロー

1. **初期化**: ユーザー設定と固有のカナリートークンをメインAIに組み込みます。
2. **入口チェック**: `classify_sensitive_content_cached` でユーザーの攻撃意図を検証。検知時は即遮断。
3. **メイン処理**: 安全な入力に対し、メインAIが回答を生成（Few-Shot耐性）。
4. **出口チェック**: 回答を `likely_leak` でスキャンし、**カナリートークンの流出**を確認。
5. **最終出力**: 
    - **トークン検出時**: 回答を破棄し、回避用AIの生成物（Status: [遮断]）を表示。
    - **安全な場合**: メインAIの回答（Status: [安全]）を表示。

## 4. 主要ファイル構成
- **`backend.py`**: コア・ロジック（LLM判定、正規化、キャッシュ、カナリー監視）。
- **`ui.py`**: 各フェーズのセキュリティオーケストレーション。
- **`app.py`**: エントリーポイント。
